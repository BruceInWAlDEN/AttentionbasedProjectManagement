系统可以看做三个对象的相互关系，这三个模块分别是：
DOCUMENT
ACTION
RECORD
每个模块都有自己的数据结构,且都可以用json格式保存。三个模块构成一个明确的树的结构，DOCUMENT->ACTION->ACTION，应该用树的数据结构来
表示这些模块之间的关系。但考虑到可扩展性，这里使用使用模块名列表和邻接矩阵来表示模块直接的关系，使用字典{“模块名”: 模块对象json字典}来保存
各个模块的信息。

模块名编码：使用模块类别和统一ID编码
模块对象数据结构：json字典
FILE类: 实现文件对象到字符串的映射
TIME类：实现时间字符串输出
模块基类：将对象所有属性生产json格式字典（包括FILE类和TIME类，其他属性），生产展示模版，基类包括的基础属性有：提交时间，ID,模块类别，检查基类属性的有效性
模块扩展类：按照三种模块的需要扩展，包含输入信息格式检查，由属性生产对应json字典，通过json字典载入属性，修改任意属性

基本接口函数：实现给定模块名，返回对应模块对象；给定模块名，以及节点信息，覆盖写入该模块对象，隐藏指定对象，

函数式编程：
实现的功能有：对每个模块进行查询，添加，删除，修改，统计
其中查询部分包括查询和显示，查询部分包括按照模块的任意参数的便利筛选，可扩展的
其中添加部分包括，获取模版数据，添加
删除部分包括，临时删除，恢复，永久删除
统计部分包括模块任意信息的统计，可扩展的
